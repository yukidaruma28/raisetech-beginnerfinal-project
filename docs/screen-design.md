## 実装済みレイアウト（2026-04-25 時点）

実装は **Linear のリストビュー風の縦アコーディオン**を採用している（旧版ではカンバン列レイアウトを想定していたが、密度の高い情報表示を優先して切り替え）。

```
┌─────────────────────────────────────────────────────────────────┐
│ Inquiry Board                                                    │  ヘッダー
├─────────────────────────────────────────────────────────────────┤
│ ▼ ● Backlog  2                                                   │  Status セクション（折りたたみ可）
│   □ [低] TASK-1   ●  ロゴデザインのリニューアル検討    Apr 25     │
│   □ [低] TASK-2   ●  問い合わせフォームの項目追加        Apr 25     │
│ ▼ ● Todo  3                                                       │
│   □ [高] TASK-3   ●  ヘッダーのナビゲーション改修        Apr 25     │
│   □ [中] TASK-4   ●  FAQ ページの作成                    Apr 25     │
│ ▶ ● In Progress  2                                                │  折りたたみ中（ChevronRight）
│ ...                                                                │
└─────────────────────────────────────────────────────────────────┘
```

**InquiryRow の構成（左から右へ）**:
1. **チェックボックス**（hover 時のみ表示）
2. **優先度バッジ**: 漢字 1 文字（高 / 中 / 低）+ priority.color の薄い背景
3. **識別子**: `TASK-{id}` 形式（例: `TASK-3`）
4. **Status カラードット**: status.color の小さな円
5. **タイトル**: 1 行 + truncate
6. **作成日**: `Intl.DateTimeFormat('ja-JP', { month: 'short', day: 'numeric' })`

### 問い合わせカード（サムネイル）レイアウト詳細

| エリア | 仕様 |
|--------|------|
| 優先度アイコン | カード左上。priorities.color + priorities.level をもとに、Linear 風のアイコン（線本数で表現）を表示 |
| カテゴリバッジ | カテゴリが設定されていれば薄いグレー枠で文字表示 |
| ラベルバッジ | 横並び表示。HEX カラーで背景色、白文字でラベル名 |

| カード全体 | クリックで InquiryModal を開く。ドラッグ可能 |

### InquiryEditDialog レイアウト（実装済み）

**保存方式: inline-edit + auto-save**（フィールドをクリック → 入力 → blur/変更で即 PATCH。保存ボタンなし）

```
┌──────────────────────────────────────────────────┐  ← オーバーレイ（クリックで閉じる）
│                                               ×  │  ← 閉じるボタン
│  タイトル                                        │
│  ┌──────────────────────────────────────────┐   │  ← クリックでテキスト入力に切替（blur で PATCH）
│  │ ログインできない                          │   │
│  └──────────────────────────────────────────┘   │
│  本文                                            │
│  ┌──────────────────────────────────────────┐   │  ← 同上
│  │ パスワードリセット後に...                 │   │
│  └──────────────────────────────────────────┘   │
│  ┌──────────────┬───────────────┐               │
│  │ ステータス   │  優先度        │               │  ← セレクト変更で即 PATCH
│  │ [Todo   ▼]   │  [高    ▼]    │               │
│  └──────────────┴───────────────┘               │
│                          ［ 🗑 削除 ］            │  ← 削除のみ（保存ボタンなし）
└──────────────────────────────────────────────────┘
```

| エリア | 仕様 |
|--------|------|
| モーダル全体 | 幅640px、最大高さ85vh、超過時はモーダル内スクロール |
| オーバーレイ | 半透明ブラック（背景）、クリックで閉じる |
| ×ボタン | 右上固定 |
| タイトル入力 | inline-edit（クリックで input に切替、blur で PATCH）、必須 |
| 本文入力 | inline-edit（クリックで textarea に切替、blur で PATCH）、任意 |
| ステータス | セレクトボックス。変更即 PATCH |
| 優先度 | セレクトボックス（3 段階固定）。変更即 PATCH |
| 担当者 / カテゴリ / ラベル | ❌ MVP スコープ外（実装しない）|
| 削除ボタン | クリックで確認ダイアログを表示し、確認後 DELETE |
| 未保存チェック | タイトル/本文の変更中にモーダルを閉じようとした場合、確認ダイアログを表示 |

## 画面状態定義

### ボード画面

| 状態 | 発生条件 | 表示内容 |
|------|---------|---------|
| ローディング | 初回データ取得中 | 列形のスケルトン3列を表示 |
| 空状態 | ステータスが0件 | 中央に「ステータスを追加してボードを始めましょう」＋追加ボタン |
| 通常 | ステータスが1件以上 | 列と問い合わせカードを表示 |
| エラー | API 取得失敗 | 右下にエラートーストを表示（ボードは空のまま） |

### ステータス列

| 状態 | 発生条件 | 表示内容 |
|------|---------|---------|
| 空状態 | 該当ステータスの問い合わせが0件 | 追加ボタンのみ表示 |
| 通常 | 問い合わせが1件以上 | 問い合わせカードを縦に並べて表示 |
| 上限到達 | 問い合わせ500件 | 「問い合わせを追加」ボタンを非表示 |

### InquiryModal

| 状態 | 発生条件 | 表示内容 |
|------|---------|---------|
| 通常 | 問い合わせクリック時 | 問い合わせの現在の内容を表示 |
| タイトル空 | タイトル入力欄が空 | 保存ボタンを非活性にする |
| 保存中 | API 呼び出し中 | 保存ボタンを disabled にする（二重送信防止） |
| 未保存差分あり | 編集後に閉じようとしたとき | 確認ダイアログ表示 |

## インタラクション仕様

| 操作 | トリガー | 挙動 |
|------|---------|------|
| 問い合わせ作成 | ヘッダーの「＋新規問い合わせ」 / 列下部の「＋」 | InquiryCreateModal が開く |
| 問い合わせ作成確定 | モーダル内「作成」ボタン | API を呼び出し、モーダルを閉じる |
| ステータス列追加 | 「＋ステータス追加」クリック | インライン入力フォーム（名前＋色）に切り替わる |
| ステータス列追加確定 | Enter | API を呼び出し、新しい列が末尾に表示 |
| ステータス列追加キャンセル | Esc | フォームが閉じる |
| ステータス名編集開始 | 列ヘッダーのタイトル部分クリック | インライン編集に切り替わる |
| ステータス名編集確定 | Enter / フォーカスアウト | API を呼び出し、入力が元表示に戻る |
| ステータス並び替え | 列ヘッダーを横方向にドラッグ | `PATCH /api/statuses/:id/move` |
| ステータス削除 | 列ヘッダーの🗑ボタン | 確認ダイアログ（移動先選択含む）→ API 呼び出し |
| 問い合わせ DnD 開始 | カードをドラッグ | ドラッグ中のカードが半透明になり、ドロップ先にプレースホルダー（灰色）を表示 |
| 問い合わせ DnD 確定 | ドロップ | `PATCH /api/inquiries/:id/move` を呼び出し、楽観的更新後にクエリ revalidate |
| 問い合わせ DnD キャンセル | 元位置にドロップ / Esc | 変化なし、API は呼び出さない |

## モーダル項目定義

| 項目 | 入力形式 | バリデーション |
|------|---------|--------------|
| タイトル | テキスト入力（1行） | 必須・255文字以内・空の場合は保存ボタン非活性 |
| 本文 | テキストエリア（6行） | 任意・文字数制限なし |
| ステータス | セレクトボックス | 必須・statuses から選択 |
| 優先度 | セレクトボックス | 任意・priorities から選択、「なし」も選択可 |
| 担当者 | テキスト入力（1行） | 任意・100文字以内 |
| カテゴリ | テキスト入力（1行） | 任意・100文字以内 |
| ラベル選択 | チェックボックス一覧 | 任意・複数選択可 |
| ラベル名（作成時） | テキスト入力 | 必須・100文字以内 |
| ラベル色（作成時） | カラーピッカー | 必須・HEX 形式 |
