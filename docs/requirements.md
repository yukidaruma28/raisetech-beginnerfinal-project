## 機能要件

### 問い合わせ管理（inquiry）
| # | 機能 | 説明 |
|---|------|------|
| I-1 | 問い合わせ作成 | タイトル・本文・カテゴリ・担当者・優先度・ステータス・ラベルを指定して問い合わせを登録する |
| I-2 | 問い合わせ表示 | ボード上のステータス列に、該当ステータスの問い合わせを縦並びで表示する |
| I-3 | 問い合わせ詳細編集 | モーダルでタイトル・本文・カテゴリ・担当者・優先度・ラベルを編集する |
| I-4 | 問い合わせ削除 | 問い合わせを削除する |
| I-5 | 問い合わせ移動（DnD） | ドラッグ＆ドロップでステータス列間・列内を移動する（ステータス変更＋並び順変更） |

### ステータス管理（status）— Linear 準拠のプロパティ方式
| # | 機能 | 説明 |
|---|------|------|
| S-1 | ステータス作成 | 名前・色を指定してステータスを作成する。ボードに新しい列として表示される |
| S-2 | ステータス編集 | 名前・色・列の並び順を編集する |
| S-3 | ステータス削除 | ステータスを削除する。所属していた問い合わせは削除防止のため、移動先ステータスを指定する |

### 優先度管理（priority）— Linear 準拠のプロパティ方式
| # | 機能 | 説明 |
|---|------|------|
| P-1 | 優先度作成 | 名前・レベル・色を指定して優先度を作成する |
| P-2 | 優先度編集 | 名前・レベル・色・並び順を編集する |
| P-3 | 優先度削除 | 優先度を削除する。所属していた問い合わせの優先度は NULL になる |

### ラベル管理（label）
| # | 機能 | 説明 |
|---|------|------|
| LB-1 | ラベル作成 | 名前と色（HEXカラー）を指定してラベルを作成する |
| LB-2 | ラベル付与 | 問い合わせにラベルを追加する（複数可） |
| LB-3 | ラベル解除 | 問い合わせからラベルを外す |
| LB-4 | ラベル削除 | ラベル自体を削除する |

## 用語定義

| 用語 | 説明 |
|------|------|
| 問い合わせ（inquiry） | 本アプリの中心的なデータ。既存 Trello アプリの「カード」に相当するが、問い合わせチケットとしての属性（カテゴリ・担当者・優先度）を持つ |
| ステータス（status） | 問い合わせの対応状態を表すプロパティ。カンバンボードの「列」として表現される。ユーザーが自由にCRUDできる |
| 優先度（priority） | 問い合わせの重要度。Linear の Urgent / High / Medium / Low / No priority のようにユーザーが定義する |
| ラベル（label） | 問い合わせに付与する分類タグ。1件の問い合わせに複数付与可能 |
| 担当者（assignee） | 問い合わせの対応担当者。認証を持たないためテキストで自由入力する |
| プロパティ | ステータス・優先度・ラベルなど、問い合わせに紐づけるメタデータの総称（Linear 由来の概念） |

## Trello 風アプリ（raisetech_kanban）との比較

本アプリは既存アプリの発展形ではなく、**別スタック・別リポジトリ・別題材**の独立プロジェクトとして開発する。比較の目的は、学習した設計パターンの再適用と差分の把握。

| 観点 | raisetech_kanban（既存） | 本アプリ（新規） |
|------|-------------------------|----------------|
| 題材 | タスク管理（Trello 風） | 問い合わせ管理（Linear 風） |
| 中心エンティティ | card（タスク） | inquiry（問い合わせ） |
| 列の意味 | ユーザー定義のリスト | ステータス（Backlog / Todo / ... のような状態） |
| 列の管理 | リストテーブルで自由に CRUD | ステータステーブルで自由に CRUD |
| 優先度 | なし | あり（プロパティとして CRUD） |
| バックエンド | Java / Spring Boot | Ruby / Rails API |
| フロントエンド | React + Vite | Next.js App Router |
| DB | PostgreSQL | MySQL |
| デプロイ | AWS EC2 + RDS | AWS EC2 + RDS（共有） |
