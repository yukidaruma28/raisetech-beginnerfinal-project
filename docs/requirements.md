## 機能要件

### 作品管理（inquiry）
| # | 機能 | 説明 |
|---|------|------|
| I-1 | 作品作成 | 作品名・メモ・優先度・視聴ステータスを指定して作品を登録する |
| I-2 | 作品表示 | ボード上の視聴ステータス列に、該当ステータスの作品を縦並びで表示する |
| I-3 | 作品詳細編集 | モーダルで作品名・メモ・視聴ステータス・気になり度を編集する |
| I-4 | 作品削除 | 作品を削除する |
| I-5 | 作品移動（DnD） | ドラッグ＆ドロップでステータス列間・列内を移動する（ステータス変更＋並び順変更） |

### ステータス管理（status）— Linear 準拠のプロパティ方式
| # | 機能 | 説明 |
|---|------|------|
| S-1 | ステータス作成 | 名前・色を指定してステータスを作成する。ボードに新しい列として表示される |
| S-2 | ステータス編集 | 名前・色・列の並び順を編集する |
| S-3 | ステータス削除 | ステータスを削除する。所属していた作品は削除防止のため、移動先ステータスを指定する |

### 優先度管理（priority）— Linear 準拠のプロパティ方式
| # | 機能 | 説明 |
|---|------|------|
| P-1 | 優先度作成 | 名前・レベル・色を指定して優先度を作成する |
| P-2 | 優先度編集 | 名前・レベル・色・並び順を編集する |
| P-3 | 優先度削除 | 優先度を削除する。所属していた作品の優先度は NULL になる |

### ラベル管理（label）
| # | 機能 | 説明 |
|---|------|------|
| LB-1 | ラベル作成 | 名前と色（HEXカラー）を指定してラベルを作成する |
| LB-2 | ラベル付与 | 作品にラベルを追加する（複数可） |
| LB-3 | ラベル解除 | 作品からラベルを外す |
| LB-4 | ラベル削除 | ラベル自体を削除する |

## 用語定義

| 用語 | 説明 |
|------|------|
| 作品（inquiry） | 本アプリの中心的なデータ。視聴したいアニメ・映画を表す。視聴ステータス・気になり度などのプロパティを持つ |
| ステータス（status） | 作品の視聴進捗を表すプロパティ。カンバンボードの「列」として表現される。ユーザーが自由にCRUDできる |
| 気になり度（priority） | 作品の視聴優先度。高 / 中 / 低 の 3 段階固定 |
| ラベル（label） | 作品に付与する分類タグ。1件の作品に複数付与可能 |
| プロパティ | ステータス・気になり度・ラベルなど、作品に紐づけるメタデータの総称（Linear 由来の概念） |

## Trello 風アプリ（raisetech_kanban）との比較

本アプリは既存アプリの発展形ではなく、**別スタック・別リポジトリ・別題材**の独立プロジェクトとして開発する。比較の目的は、学習した設計パターンの再適用と差分の把握。

| 観点 | raisetech_kanban（既存） | 本アプリ（新規） |
|------|-------------------------|----------------|
| 題材 | タスク管理（Trello 風） | 視聴管理（Linear 風） |
| 中心エンティティ | card（タスク） | inquiry（作品） |
| 列の意味 | ユーザー定義のリスト | ステータス（Backlog / Todo / ... のような状態） |
| 列の管理 | リストテーブルで自由に CRUD | ステータステーブルで自由に CRUD |
| 優先度 | なし | あり（プロパティとして CRUD） |
| バックエンド | Java / Spring Boot | Ruby / Rails API |
| フロントエンド | React + Vite | Next.js App Router |
| DB | PostgreSQL | MySQL |
| デプロイ | AWS EC2 + RDS | AWS EC2 + RDS（専用、Terraform 完全管理） |
